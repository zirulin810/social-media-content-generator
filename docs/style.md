# 視覺風格與內容紅線 v1.0

> 圖卡與文案的單一事實來源。`templates/base.css` 直接吃這裡的 token，**程式裡不准出現硬編色碼**。
> 決策過程見 Obsidian 的 [[視覺風格與內容紅線]]。

## 風格：品牌色塊（白底墨綠）

白底、墨綠色塊。挑這個是因為 IG 滑動時可讀性最高、不挑字長，色塊讓連發多張時有一致的識別度。

## Design Token

```css
:root {
  /* 顏色 */
  --color-bg:        #FFFFFF;  /* 卡片底色 */
  --color-accent:    #0F6E63;  /* 墨綠：頂部色塊、作者名、封面主色 */
  --color-text:      #12211F;  /* 金句主文 */
  --color-text-mute: #5B6B68;  /* 次要資訊（handle、出處小字） */
  --color-border:    #E5E5E5;  /* 卡片外框 */

  /* 字體：繁中優先，缺字才 fallback */
  --font-sans: "Noto Sans TC", "Source Han Sans TC", "PingFang TC",
               "Microsoft JhengHei", sans-serif;

  /* 字級（以 1080px 寬的圖卡為基準） */
  --size-quote:   72px;  /* 金句主文；長句可自動降到 --size-quote-min */
  --size-quote-min: 52px;
  --size-title:   64px;  /* 封面標題 */
  --size-author:  32px;  /* 原作者標註 */
  --size-meta:    28px;  /* handle、出處小字 */

  /* 排版 */
  --line-height-quote: 1.45;
  --letter-spacing:    0.02em;
  --weight-quote:      700;

  /* 間距 */
  --pad-card:   96px;  /* 卡片四周安全邊距，重要文字不貼邊 */
  --bar-height: 48px;  /* 頂部墨綠色塊 */
  --radius:      0px;  /* 直角，不做圓角 */
}
```

## 版面規則

- **頂部墨綠色塊**（`--bar-height`）是三種卡片共用的識別元素，不可省。
- **金句垂直置中**，原作者標註靠左下。
- 長金句（接近 40 字）自動降字級到 `--size-quote-min`，不可溢出、不可裁切。
- 四周留 `--pad-card` 安全邊距——IG 縮圖與某些裝置會吃掉邊緣。

## 標註規則（版權紅線，不可省）

| 元素 | 封面卡 | 金句卡 | 結尾卡 |
| --- | --- | --- | --- |
| 原作者名 | ✓ | ✓ | ✓ |
| 我的 handle | ✓ | ✗（保持乾淨） | ✓ |
| 原影片連結 | ✗ | ✗ | ✓ |

- caption 結尾一律放**完整出處**：影片標題 + 作者 + 原影片連結。
- 出處指回**原影片**，不是我的剪報檔。

## 內容紅線

1. **不扭曲原意。** 可潤飾字句、可翻譯，不可斷章取義改變作者的主張，不可加料。
2. **不做標題黨。** 不用「震驚」「99% 的人不知道」「再不看就晚了」——**誇大與製造焦慮才是標題黨**。
3. **不捏造任何人的經驗。** 不編造「我」的過去，也不編造「你」的過去。
   要寫「你踩過的坑」，那個坑必須是**知識卡裡真的有的**（通常是對照卡的 ✗ 那面）。
4. **不省出處。** 任何一則貼文都必須看得到原作者與原連結。
5. **不假裝原創。** 誠實呈現「這是我讀別人的東西之後的整理」，作者的功勞是作者的。
6. **不用表情符號。** 圖卡與 caption 皆不用。

## 開場白（hook）

**hook 的唯一任務：讓人停止滑動。** 它是貼文裡唯一一定會被讀到的東西
（IG 折疊前只顯示 125 字；Threads 只給一行）。

**hook 是獨立欄位，不是「正文的第一句」。** 規則：

- **30 字以內，一句話。** 太長就不是 hook，是摘要。
- **不准是 `angle` 的複述**（程式會驗字元重疊率）。
	- `angle` ＝「這則在講什麼」（給我看的索引，也是封面標題）
	- `hook` ＝「**你為什麼該停下來**」（給讀者看的）
	- ✗「用『槓鈴策略』駕馭 AI，兼顧防禦與進攻，讓 AI 成為你的思考夥伴」——**沒有人會為一句摘要停下手指**
- **形態**：痛點／反直覺／結果／共鳴提問。依據要在知識卡裡找得到。
- **hook 是承諾，正文要兌現。** 不准丟一個這則貼文答不出來的問題。

## 文案語氣（v1.1 改）

**第二人稱。** 焦點在讀者身上，不在「我」身上。預設 **`ask`（提問型開場）**。

實作：`prompts/voice_ask.md` / `voice_claim.md`（切換：`COMPOSE_VOICE=claim`）。

> **命名的教訓**：這兩個變體原本叫 `hook` / `direct`——好像「用問句才叫 hook」。
> **那是錯的：兩種都是 hook，差別只在句型。** 名字騙人，思考就會跟著歪掉。

### 為什麼把「不用問句鉤讀者」這條紅線拿掉

v1.0 寫著「不用問句鉤讀者（『你有沒有發現…』），那會滑向標題黨」。**那條規則抓錯了地方。**

Human 2026-07-14 的反駁：「我不覺得用問句鉤讀者算是一種標題黨，**如果讀者真的有相關經驗呢**？」

他是對的。**標題黨的定義是「誇大或不實」，不是「用了問句」。**
如果原文真的講了「很多人一開始就把舊筆記全部匯入，結果什麼都找不到」，
那句問句只是**換個人稱在講一件真事**，讀者的共鳴是真的。

所以規則從「禁句型」改成「禁捏造」——**修辭可以，捏造不行**，
這跟整條 pipeline 的 grounding 原則是同一條。

（第一版的第一人稱語氣還有一個副作用：模型會**替使用者編造個人經歷**。
「我一直以為筆記要分類分得很細，看完才發現方向反了」被它套進四則不同主題的貼文，
包括跟分類毫無關係的那幾則——**那是一段沒發生過的心路歷程**，而且是我在 prompt 裡示範給它的。）
