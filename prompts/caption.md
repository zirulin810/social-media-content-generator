# `hook`：開場白（**最重要的欄位**）

**hook 的唯一任務：讓人停止滑動。** 它是這則貼文唯一保證會被讀到的東西
（IG 折疊前只顯示前 {fold} 字；Threads 只給你一行）。

- **{hook_max} 字以內，一句話。** 太長就不是 hook，是摘要。
- **不准是 `angle` 的複述**（程式會驗）。angle 是「這則在講什麼」，hook 是「**你為什麼該停下來**」。
	- ✗「用『槓鈴策略』駕馭 AI，兼顧防禦與進攻，讓 AI 成為你的思考夥伴。」（**那是摘要，沒有人會為它停下手指**）
- **不准出現標題與作者名**（出處由程式接在文末）。
- **hook 是承諾，正文要兌現它。** 不要問一個這則貼文答不出來的問題。

## hook 的八種手法（挑最貼合這則內容的一種）

| 手法 | 例子 |
| --- | --- |
| **痛點情境** | 「AI 每次都要你重講一遍你是誰。」 |
| **反直覺** | 「分類分得越細，越找不到東西。」 |
| **具體結果** | 「17,000 則筆記，AI 只讀三個檔案就上手。」 |
| **共鳴提問** | 「你是不是也把舊筆記全部匯進來了？」 |
| **代價／取捨** | 「技能存在 AI 工具裡，換一次工具就全部歸零。」 |
| **破除誤解** | 「Obsidian 沒有內建 AI。這是刻意的。」 |
| **門檻很低** | 「三個檔案，就能讓 AI 讀懂一萬則筆記。」 |
| **作者原話** | 直接引用金句卡那句話（如果它本身夠有力） |

**每一種的依據都必須在上面的卡片裡找得到。** 卡片沒講的，不要編。

## ⛔ hook 的紅線：**不准替讀者斷言他的狀態**

- ✗「你的筆記散落在各處，**從來沒有真正屬於你**。」
- ✗「你的筆記之所以難用，**是因為你沒有真正擁有它**。」
- ✗「你一直用錯方法。」

**這些句子是在替讀者的人生下判斷——而你根本不知道他的狀況。**
它們假裝了解他的缺陷，然後叫他來看解方。**那叫戴帽子，不叫共鳴。**

差別在哪：

- 「你是不是也把舊筆記全部匯進來了？」→ **問句，他可以說沒有**（而且原文真的講過這是常見錯誤）
- 「你的筆記從來沒有真正屬於你」→ **斷言，他沒有否認的餘地**，而且很可能不是事實

**講內容，講原文講過的常見錯誤，講一個開放的問題。不要替他定義他是誰。**

# `body`：正文

- **3 到 5 句話。就這樣。**
  不是「大約幾百字」——是**三到五句**。你（跟所有語言模型一樣）**不會數中文字**，所以用句數。
- **一句一行**（句子之間換行）。不要黏成一整塊。
- **第一句先兌現 hook 的承諾**，不要重複 hook 那句話。
- 每一句都能單獨成立。**寫完第五句就停手**——不要補結論、不要補呼籲、不要寫「總之」。
- **不要**寫出處或連結（程式會接）。**不要**放 hashtag（另外給）。
- **不准出現影片／文章的標題與作者名。**

# 紅線（違反等於整篇重寫）

1. **不扭曲原意**：**不得超出上面那些卡片講過的東西。**
2. **不做標題黨**：不用「震驚」「99% 的人不知道」「再不看就晚了」——**誇大與製造焦慮才是標題黨**。
3. **不捏造任何人的經驗**：不編造「我」的過去，也不編造「你」的過去。
	- ✗「我一直以為筆記要分類分得很細，看完才發現方向反了」（**編造我的過去**）
	- ✗「你是不是也覺得筆記很難整理？」（卡片沒講過 —— **憑空編讀者的經驗**）
4. **不替讀者斷言**（見上面的 hook 紅線）。
5. **不假裝原創**：這是讀完 {kind} 之後的整理，作者的功勞是作者的。
6. **不用表情符號。** 一個都不行。
7. **一律台灣繁體中文**，不用簡體字、不用中國用語（視頻→影片、軟件→軟體、文件夾→資料夾）。

# 語氣

**第二人稱。焦點在讀者和方法上，不在「我」身上。**
「我」只在必要時出現（標明這是轉述）。讀者不在乎我怎麼樣，他在乎這對他有沒有用。

# hashtag（只有 IG 會用到）

5–10 個，繁體中文或英文皆可。**要跟內容有關**，不要通用垃圾標籤（#每日一句 #成長）。

# 輸出格式

只輸出 JSON，不要解釋、不要 markdown 圍欄：

```
{
  "hook": "開場白（{hook_max} 字內，一句話）",
  "body": "第一句。\n第二句。\n第三句。",
  "hashtags": ["#Obsidian", "#筆記術"]
}
```
