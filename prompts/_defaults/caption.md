# `hook`：開場白（**最重要的欄位**）

**hook 的唯一任務：讓人停止滑動。** 它是這則貼文唯一保證會被讀到的東西
（IG 折疊前只顯示前 {fold} 字；Threads 只給你一行）。

- **一句話。就一句。**（程式會檢查：不能換行、不能有兩個句號）
	- 判準**不是字數**，是「**它只講一個意思嗎**」。一句話講兩件事，那就是摘要。
	- 大概 {hook_max} 字上下最有力，但那只是參考——**短而有力，勝過剛好卡在某個字數。**
- **不准是 `angle` 的複述**（程式會驗）。angle 是「這則在講什麼」，hook 是「**你為什麼該停下來**」。
	- ✗「用『槓鈴策略』駕馭 AI，兼顧防禦與進攻，讓 AI 成為你的思考夥伴。」（**那是摘要，沒有人會為它停下手指**）
- **不准出現標題與作者名**（出處由程式接在文末）。
- **hook 是承諾，正文要兌現它。** 不要問一個這則貼文答不出來的問題。

## hook 的八種手法（挑最貼合這則內容的一種）

| 手法 | 例子 |
| --- | --- |
| **痛點情境** | 「AI 每次都要你重講一遍你是誰。」 |
| **反直覺** | 「分類分得越細，越找不到東西。」 |
| **具體結果** | 「17,000 則筆記，AI 只讀三個檔案就上手。」 |
| **共鳴提問** | 「你是不是也把舊筆記全部匯進來了？」 |
| **代價／取捨** | 「技能存在 AI 工具裡，換一次工具就全部歸零。」 |
| **破除誤解** | 「Obsidian 沒有內建 AI。這是刻意的。」 |
| **門檻很低** | 「三個檔案，就能讓 AI 讀懂一萬則筆記。」 |
| **作者原話** | 直接引用金句卡那句話（如果它本身夠有力） |

**每一種的依據都必須在上面的卡片裡找得到。** 卡片沒講的，不要編。

## ⛔ hook 的紅線：**不准替讀者斷言他的狀態**

- ✗「你的筆記散落在各處，**從來沒有真正屬於你**。」
- ✗「你的筆記，**是不是從來沒有真正屬於你**？」（**加問號不代表沒戴帽子**——
  它一樣預設了「你的筆記不屬於你」這件事，讀者只是被邀請承認它）
- ✗「你的筆記之所以難用，**是因為你沒有真正擁有它**。」
- ✗「你一直用錯方法。」

**這些句子是在替讀者的人生下判斷——而你根本不知道他的狀況。**
它們假裝了解他的缺陷，然後叫他來看解方。**那叫戴帽子，不叫共鳴。**

差別在哪：

- 「你是不是也把舊筆記全部匯進來了？」→ **問句，他可以說沒有**（而且原文真的講過這是常見錯誤）
- 「你的筆記從來沒有真正屬於你」→ **斷言，他沒有否認的餘地**，而且很可能不是事實

**講內容，講原文講過的常見錯誤，講一個開放的問題。不要替他定義他是誰。**

# `body`：正文

## ⛔ **不要當解說員。讀者看不到 {kind}。**

他滑到的是**你的輪播圖**，不是 {kind}。他不會為了看懂這段文字去點開一支 40 分鐘的影片。

所以：**直接跟他講那個想法本身**，不要幫一個他看不到的東西做導覽。

- ✗「**這支影片展示了**如何用 AI 技能系統自動化工作流。」
- ✗「**影片建議**，一開始保持簡單，專注於連結筆記。」
- ✗「**就像作者所說**，Obsidian 只是看著你電腦上的一個資料夾。」
- ✗「**這支影片點出**新手常見的四大錯誤。」

上面每一句都在講「那支影片說了什麼」。**讀者要的是「這件事是什麼、對我有什麼用」。**

- ✓「AI 技能系統可以幫你自動化重複的雜事——每天早上的簡報就是一個例子。」
- ✓「一開始把外掛裝好裝滿，反而會分散你學核心功能的注意力。」
- ✓「Obsidian 只是看著你電腦上的一個資料夾。你的筆記從頭到尾都在你手上。」

**出處由程式接在文末**（「原文：《標題》｜作者 + 連結」）——**誠實已經由那一行負責了**，
你不需要在每句話裡再提醒讀者「這是別人講的」。

## 寫法

- **正文總共不超過 {body_max} 字**（程式會檢查）——但那是天花板，**不是目標**。
	- 目標是下面那條：2–3 段、每段 ≤{para_max} 字。
	- **寧可少講一件事，也不要縮寫成流水帳。** 講不完的東西，圖卡裡有。
- **寫成 2 到 3 個小段落，每段 2 到 3 句、大約 {para_max} 字。**
	- **判準是句數，不是字數**（你數不準中文字）。一段 2–3 句，就是一段。
	- 段落之間**空一行**（`\n\n`）。
	- **不要把整篇擠成一坨**——實跑時你寫過一段 300 字的，讀者直接滑走。
- **句子要連起來，不要一句一行。**
  一句一行會變成「沒有項目符號的條列」——**每句都在講一件不相干的事，讀起來東落西落。**
  用「因為」「所以」「但是」「而這代表」把它們接起來，讓它像一段話，不像一張清單。
- **第一段先兌現 hook 的承諾**，不要重複 hook 那句話。
- **正文不要再鉤一次讀者。** hook 已經做過了。
	- ✗ 正文出現「你是不是也覺得知識很難累積？」「想像一下…」
	- 正文的任務是**兌現 hook 的承諾**——把那件事講清楚。
- **寫完第三段就停手**——不要補結論、不要補呼籲、不要寫「總之」。
- **不要**寫出處或連結（程式會接）。**不要**放 hashtag（另外給）。
- **不准出現 {kind} 的標題與作者名。**

# 紅線（違反等於整篇重寫）

1. **不扭曲原意**：**不得超出上面那些卡片講過的東西。**
2. **不做標題黨**：不用「震驚」「99% 的人不知道」「再不看就晚了」——**誇大與製造焦慮才是標題黨**。
3. **不捏造任何人的經驗**：不編造「我」的過去，也不編造「你」的過去。
	- ✗「我一直以為筆記要分類分得很細，看完才發現方向反了」（**編造我的過去**）
	- ✗「你是不是也覺得筆記很難整理？」（卡片沒講過 —— **憑空編讀者的經驗**）
4. **不替讀者斷言**（見上面的 hook 紅線）。
5. **不當解說員**：不寫「這支影片…」「影片中提到…」「作者建議…」——**讀者看不到它。**
	（誠實由文末的出處負責，不必在句子裡一直提醒。）
6. **不假裝原創**：作者的功勞是作者的——那由文末的出處標註完成，不是靠在正文裡一直複述。
6. **不用表情符號。** 一個都不行。
7. **一律台灣繁體中文**，不用簡體字、不用中國用語（視頻→影片、軟件→軟體、文件夾→資料夾）。

# 語氣

**第二人稱。焦點在讀者和方法上，不在「我」身上。**
「我」只在必要時出現（標明這是轉述）。讀者不在乎我怎麼樣，他在乎這對他有沒有用。

# hashtag（只有 IG 會用到）

5–10 個，繁體中文或英文皆可。**要跟內容有關**，不要通用垃圾標籤（#每日一句 #成長）。

# 輸出格式

只輸出 JSON，不要解釋、不要 markdown 圍欄：

```
{
  "hook": "開場白（{hook_max} 字內，一句話）",
  "body": "第一句。\n第二句。\n第三句。",
  "hashtags": ["#Obsidian", "#筆記術"]
}
```
