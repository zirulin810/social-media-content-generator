# 自動化輸出

把一篇文章（本機 markdown）變成**可直接發布的 IG／Threads 貼文**：知識卡 → 社群圖卡 → 文案 → 半自動發布。全程在你自己的電腦上跑，一個本機編輯台搞定。

```
素材 .md 拖進編輯台 → AI 抽知識卡＋出圖＋寫文案 → 你直接在卡片上改字 → 模擬預覽 → 發布
```

核心理念：**文字是可以直接編輯的東西，不是 prompt 的產物。** AI 負責起草，人直接改字定稿——不用改 prompt 重跑碰運氣。

## 功能

- **編輯台**（本機網頁）：素材拖進來就跑完整條 pipeline；圖卡上每一個字直接點著改；卡片可增刪排序；五種主題、1:1／4:5 即時切換
- **AI 只提議、不覆蓋**：改短、換 hook、重新起草都以並排差異呈現，逐條接受或拒絕；人改過的內容 AI 永遠不會覆蓋（資料層強制，有測試釘死）
- **忠於原文**：知識卡的每一條主張都帶原文依據，程式逐條核對；出處印在每組圖的結尾卡上，關掉出處必須明確宣告原創
- **IG／Threads 模擬**：發文前看到真實長相——IG 125 字折疊、Threads 500 字上限、輪播順序
- **半自動發布**：瀏覽器自動填好圖與文案，**最後的「分享」由你按**；另可一鍵輸出圖＋文案到本地資料夾當保險
- **後台設定**：密度檔位（附即時示意卡）、AI 提示詞直接編輯（含 Markdown 預覽與回復預設）、模型與 API 管理

## 需求

- Windows（內建 Edge 即可，**不會下載瀏覽器**）
- Python 3.10+
- Gemini API key（[免費申請](https://aistudio.google.com/apikey)；也支援 Anthropic）

## 上手

1. **雙擊 `開始.bat`**——第一次會自動安裝環境，然後開出編輯台
2. **貼上 API key**——編輯台右上 ⚙ 設定 → 模型與 API（或複製 `.env.example` 為 `.env` 填入）
3. 把一篇 `.md` 拖進視窗，開始

> 素材格式：一般 markdown 即可；frontmatter 需有 `source:`（原文網址——出處是紅線，沒有來源不收）。用 Obsidian Web Clipper 剪下來的文章直接可用。

## 專案結構

```
開始.bat              唯一入口：第一次自動安裝，之後直接開編輯台
tools/                開發者工具（版型校準、密度比較、煙霧測試）
src/                  pipeline 四階段：ingest → analyze → render → compose
  editor.py           編輯台資料層（人的編輯如何落地）
  publish_web.py      半自動發布（程式永遠不按「分享」——那顆鍵是你的）
  settings.py         所有可調參數的單一事實來源
templates/            圖卡版型（card.css 五主題）＋編輯台前端
prompts/              AI 提示詞（可在設定頁直接編輯；_defaults/ 是出廠版）
schemas/              資料契約（JSON Schema）——每階段產物先驗證再落地
scripts/              伺服器與開發工具（校準、審稿表、密度比較）
tests/                230+ 條測試：契約、紅線、拆卡、編輯防線全部釘死
docs/spec.md          資料契約與設計決策全紀錄（含每個數字是怎麼量出來的）
```

## 設計原則（讀 docs/spec.md 前的速覽）

> **機械的事程式做，判斷的事模型做，品味的事人來決定。**

- 三種上限住三個地方：**編輯目標**在 prompt（超過沒事）、**物理極限**在 schema（超過退回）、**密度**交給渲染器自動拆卡
- 產物新鮮度跟**每一個輸入**比：資料、prompt、版型、設定、程式碼——改了任何一個，舊產物自動過期重生
- 機器不替人判斷品質：驗得出來的（字數、結構、簡繁）就驗，驗不出來的（好不好看、有沒有超譯）標給人看

## 批次與進階

```
python -m src.cli "路徑\文章.md"          # 不開編輯台，跑完整條 pipeline
python -m src.cli "資料夾\*.md" --force   # 整批重跑
python scripts/analyze_all.py             # 批次分析＋主張↔原文的並排審稿表
python -m pytest -q                       # 跑測試
```
