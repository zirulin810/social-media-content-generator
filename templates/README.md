# templates/ — 社群圖卡版型

**版型的唯一存放處。** 改這裡，所有圖卡跟著變。

| 檔案 | 是什麼 |
| --- | --- |
| `card.css` | 兩個主題的樣式（A 編輯大字 / B 深色螢光）+ 六種卡型的排版 |
| `card.js` | 卡片渲染 + **自動適配字級**（二分搜尋） |
| `card.html` | 單張卡的容器。渲染器載入它 → 注入資料 → 截圖 |
| `preview.html` | **預覽頁**：兩個主題、兩種尺寸、含滿版壓力測試 |

## 怎麼看

雙擊專案根目錄的 **`預覽版型.bat`**（會開一個本機小伺服器，因為瀏覽器不讓 `file://` 讀 JSON）。

或直接雙擊 `card.html` 看單張示範卡（不需要伺服器）。

## 設計原則

**內容決定卡片，不是卡片決定內容。**

`card.css` 裡**沒有寫死任何字級**——全部是 `calc(var(--fs) * n)`。`--fs` 由 `card.js` 的 `autofit()` 二分搜尋出來：塞得下的最大字級。

- 內容少 → 字大（最大 76px）
- 內容多 → 字小（最小 **34px**，可讀性下限）
- 連 34px 都塞不下 → 回報 `overflow`，**由渲染器拆成兩張卡**，不砍內容

字級下限是唯一的紅線：低於它就不叫圖卡，叫掃描件。

## 換主題

`<body data-theme="a">` 或 `"b"`。尺寸是 `data-ratio="1x1"` / `"4x5"`。

## 六種卡

`cover` / `point` / `steps` / `contrast` / `quote` / `outro`

**新增卡型時，三個地方要一起改**：`card.js` 的 `TEMPLATES`、`card.css` 的樣式、
以及 `src/analyze/grounding.py` 的 `iter_claims()`——漏了最後一個，那張卡就會靜默繞過原文對照。

## 重點標記

卡片文字裡可以用 `**這樣**` 標重點：

- 主題 A → 變成磚紅色字
- 主題 B → 變成螢光筆劃過

模型在 prompt 裡就可以直接用。
